Moku:Lab plugin¶

The PLACE module for the Moku:Lab.

Currently, only a frequency response analyzer is implemented, but the Moku:Lab instrument has much greater functionality than this. Further features can be added to this module as required.

Note the MokuLab plugin requires the following information to be present in .place.cfg:

time_safety_net = enter_value_here  #(e.g. 3.0)
ip_address = enter_value here       #(e.g. 192.168.1.20)

Note

PLACE will usually add the instrument class name to the heading. For example, ch1_magnitude_data will be recorded as MokuLab-ch1_magnitude_data. The reason for this is because NumPy will not check for duplicate heading names automatically, so prepending the class name greatly reduces the likelihood of duplication.
